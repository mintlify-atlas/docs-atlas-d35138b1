## What is RedEye?

RedEye is an open-source analytic tool developed by [CISA](https://www.cisa.gov/) and [DOE](https://www.energy.gov/)'s [Pacific Northwest National Laboratory](https://www.pnnl.gov/) to assist Red Teams with visualizing and reporting Command & Control (C2) activities.

RedEye parses logs from C2 frameworks such as Cobalt Strike, Brute Ratel, and Sliver, then presents the data in a network graph that operators can annotate, tag, and present to stakeholders.

## Two modes, one binary

RedEye ships as a single binary that operates in two distinct modes, covering the complete Red Team engagement lifecycle.

| Mode | Purpose | Authentication |
|------|---------|----------------|
| **Red Team** | Upload logs, explore data, annotate findings, create presentations, export campaigns | Password required |
| **Blue Team** | Read-only review of exported campaign files | No password (default mode) |

## Supported C2 frameworks

- **Cobalt Strike** — Log folder upload (single-server or multi-server)
- **Brute Ratel** — Log folder upload
- **Sliver** — Log folder upload
- **.redeye / .sqlite** — Previously exported campaign databases

<Note>
  RedEye is currently in **maintenance mode**. CISA reviews community issues and pull requests for bug fixes, but no new features are being added.
</Note>

## Getting started

<Steps>
  <Step title="Download the binary">
    Get the latest release for your OS from the [GitHub Releases](https://github.com/cisagov/RedEye/releases) page
  </Step>
  <Step title="Choose a mode">
    Run in Red Team mode with `--redTeam --password <password>`, or double-click for Blue Team mode
  </Step>
  <Step title="Open the web app">
    Navigate to `http://127.0.0.1:4000` in your browser
  </Step>
  <Step title="Import a campaign">
    Upload C2 log files or try the [example datasets](/development/example-datasets)
  </Step>
</Steps>
