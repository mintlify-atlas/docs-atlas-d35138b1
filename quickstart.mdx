title: "Quickstart"
description: "Get RedEye running and import your first campaign in minutes"
---

RedEye runs as a local server binary. You download it, pick a mode, start the server, and open the web app in your browser. The whole process takes under five minutes.

<Steps>
  <Step title="Download the binary">
    Go to the [RedEye Releases page](https://github.com/cisagov/RedEye/releases) and download the latest binary for your operating system.

    The download contains two items:

    - The `RedEye` application binary
    - A `parsers` folder with log parser binaries (e.g. `cobalt-strike-parser`)

    Place both in the same directory before running.

    <Warning>
      **macOS only:** The first time you run RedEye, macOS may show a "not verified" error because the binary is from an unidentified developer. To allow it, open **System Preferences** > **Security & Privacy** > **General** and click **Open Anyway**. See the [Apple support page](https://support.apple.com/guide/mac-help/open-a-mac-app-from-an-unidentified-developer-mh40616/) for details.
    </Warning>
  </Step>

  <Step title="Choose a mode">
    RedEye has two modes that cover two stages of a Red Team engagement.

    <Columns cols={2}>
      <Card title="Red Team mode" icon="shield-halved">
        Full feature set: upload C2 logs, explore data, tag activity, and create presentations. Requires a password at startup. Use this to ingest and annotate campaign data.
      </Card>
      <Card title="Blue Team mode" icon="eye">
        Simplified, read-only UI for reviewing campaigns exported by a Red Team. No password required. This is the default mode.
      </Card>
    </Columns>

    <Note>
      Both modes run from the same `RedEye` binary. You switch between them using the `--redTeam` flag.
    </Note>
  </Step>

  <Step title="Start the server">
    Open a terminal in the directory containing the `RedEye` binary and run the appropriate command for your mode.

    <Tabs>
      <Tab title="Red Team mode">
        ```bash
        ./RedEye --redTeam --password <your_password>
        ```

        Replace `<your_password>` with a password of your choice. You will use this password to log in to the web app.
      </Tab>
      <Tab title="Blue Team mode">
        ```bash
        ./RedEye
        ```

        You can also double-click the `RedEye` executable to start it without opening a terminal.
      </Tab>
    </Tabs>

    The server starts and automatically opens the web app in your default browser. To stop RedEye, close the terminal window.

    <Tip>
      You can configure the server using a `config.json` file placed next to the `RedEye` binary instead of passing flags every time:

      ```json
      {
        "password": "your_password",
        "redTeam": true,
        "parsers": ["cobalt-strike-parser"]
      }
      ```
    </Tip>
  </Step>

  <Step title="Open the web app">
    Navigate to [http://127.0.0.1:4000](http://127.0.0.1:4000) in your browser. RedEye serves the UI on port 4000 by default.

    If you started the server in **Red Team mode**, you will be prompted to log in using the password you provided at startup.
  </Step>

  <Step title="Import example data">
    To explore RedEye's features right away, import the included example dataset.

    1. Download [gt.redeye](https://github.com/cisagov/RedEye/raw/develop/applications/redeye-e2e/src/fixtures/gt.redeye) from the fixtures folder.
    2. In the web app, click **+ Add Campaign**.
    3. Upload the `gt.redeye` file.

    RedEye will parse and display the campaign data. You can then explore hosts, beacons, commands, and timelines.

    Additional example datasets are available in the [fixtures folder](https://github.com/cisagov/RedEye/tree/develop/applications/redeye-e2e/src/fixtures):

    - **gt.redeye** — the primary example dataset
    - **smalldata.redeye** — a smaller dataset for quick exploration
    - **testdata** — available as raw Cobalt Strike logs
  </Step>
</Steps>

## Next steps

Once you have the example data loaded, you can:

- Tag commands and add analyst comments
- Use presentation mode to walk stakeholders through findings
- Export the campaign as a `.redeye` file for Blue Team handoff

For a full walkthrough of every feature, see the [User Guide](https://github.com/cisagov/RedEye/blob/develop/docs/UserGuide.md).
