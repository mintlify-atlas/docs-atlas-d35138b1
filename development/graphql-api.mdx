## Endpoint

The GraphQL API is available at:

```
http://127.0.0.1:4000/graphql
```

The schema is generated by [TypeGraphQL](https://typegraphql.com/) from the TypeScript resolver classes in `applications/server/src/`. The canonical schema file is `applications/server/schema.graphql`.

## Authentication

RedEye uses cookie-based authentication. In Red Team mode the server requires a password. Before making authenticated requests, obtain a session cookie by calling the login REST endpoint:

```bash
POST /api/login
Content-Type: application/json

{ "password": "<your_password>" }
```

On success the server returns a `200` response and sets an `httpOnly` cookie named `REDEYE`. Include this cookie in subsequent GraphQL requests. All GraphQL requests must be sent with `credentials: 'include'` (or the equivalent `Cookie` header) when the server is running in Red Team mode.

Blue Team mode does not require authentication.

## Core types

### Campaign

A `Campaign` is the top-level container for all imported C2 data.

```graphql
type Campaign {
  id: String!
  name: String!
  annotationCount: Int!
  beaconCount: Int!
  commandCount: Int!
  computerCount: Int!
  serverCount: Int!
  firstLogTime: DateTime
  lastLogTime: DateTime
  parsingStatus: ParsingStatus!
  migrationError: Boolean!
  parsers: [CampaignParser!]
  creator: GlobalOperator
  lastOpenedBy: GlobalOperator
}
```

### Beacon

A `Beacon` represents an active C2 implant session on a host.

```graphql
type Beacon {
  id: String!
  beaconName: String!
  displayName: String
  hidden: Boolean
  commandsCount: Float!
  commentsCount: Float!
  logsCount: Float!
  serverId: String!
  host: Host
  meta: [BeaconMeta!]!
  mitreTechniques: [MitreTechniques]
}
```

### Host

A `Host` represents a compromised machine.

```graphql
type Host {
  id: String!
  hostName: String!
  displayName: String
  hidden: Boolean
  commandsCount: Float!
  commentCount: Float!
  cobaltStrikeServer: Boolean
  beaconIds: [String!]!
  meta: [HostMeta!]!
}
```

### Annotation

An `Annotation` is a comment attached to one or more commands.

```graphql
type Annotation {
  id: String!
  text: String!
  user: String!
  date: DateTime!
  favorite: Boolean!
  commandCount: Float!
  tagCount: Float!
  tags: [Tag!]
  commandGroupId: String
  commandIds: [String]
  beaconIdFromFirstCommand: String
  generation: GenerationType!
}
```

### Operator

An `Operator` represents a Red Team member who ran commands during a campaign.

```graphql
type Operator {
  id: String!
  name: String!
  startTime: DateTime
  endTime: DateTime
  beaconIds: [String!]!
  logIds: [String!]!
  commentsCount: Float!
}
```

## Queries

### Campaign queries

```graphql
# List all campaigns
query GetCampaigns {
  campaigns {
    id
    name
    beaconCount
    commandCount
    parsingStatus
    firstLogTime
    lastLogTime
  }
}

# Get a single campaign by ID
query GetCampaign($campaignId: String!) {
  campaign(campaignId: $campaignId) {
    id
    name
    annotationCount
    beaconCount
    serverCount
    computerCount
  }
}
```

### Beacon queries

```graphql
# Get all beacons in a campaign (excluding hidden)
query GetBeacons($campaignId: String!) {
  beacons(campaignId: $campaignId, hidden: false) {
    id
    beaconName
    displayName
    commandsCount
    host {
      hostName
    }
    meta {
      startTime
      endTime
      type
      username
      ip
      pid
      process
    }
    mitreTechniques
  }
}
```

### Host queries

```graphql
# Get all hosts in a campaign
query GetHosts($campaignId: String!) {
  hosts(campaignId: $campaignId, hidden: false) {
    id
    hostName
    displayName
    commandsCount
    commentCount
    beaconIds
    meta {
      ip
      os
      osVersion
    }
  }
}
```

### Annotation and command queries

```graphql
# Get all annotations for a campaign
query GetAnnotations($campaignId: String!) {
  annotations(campaignId: $campaignId) {
    id
    text
    user
    date
    favorite
    tags {
      text
    }
    commandCount
  }
}

# Search annotations by text
query SearchAnnotations($campaignId: String!, $searchQuery: String!) {
  searchAnnotations(campaignId: $campaignId, searchQuery: $searchQuery) {
    id
    text
    user
    date
    commandIds
  }
}

# Get commands for a beacon
query GetCommandsByBeacon($campaignId: String!, $beaconId: String!) {
  commands(campaignId: $campaignId, beaconId: $beaconId) {
    id
    inputText
    commandFailed
    operator {
      name
    }
    input {
      dateTime
      blob
    }
    output {
      blob
      lineType
    }
  }
}
```

### Operator queries

```graphql
# Get all operators for a campaign
query GetOperators($campaignId: String!) {
  operators(campaignId: $campaignId) {
    id
    name
    startTime
    endTime
    commentsCount
  }
}
```

### Timeline query

```graphql
# Get a bucketed timeline of beacon activity
query GetTimeline($campaignId: String!) {
  timeline(campaignId: $campaignId, suggestedBuckets: 100) {
    campaignStartTime
    campaignEndTime
    bucketMinutes
    buckets {
      bucketStartTime
      bucketEndTime
      activeBeacons
      createdBeacons
      dyingBeacons
      deadBeacons
    }
  }
}
```

## Mutations

### Campaign mutations

```graphql
# Create a new campaign
mutation CreateCampaign($name: String!, $parser: String!, $creatorName: String!) {
  createCampaign(name: $name, parser: $parser, creatorName: $creatorName) {
    id
    name
  }
}

# Rename a campaign
mutation RenameCampaign($campaignId: String!, $name: String!) {
  renameCampaign(campaignId: $campaignId, name: $name) {
    id
    name
  }
}

# Delete a campaign
mutation DeleteCampaign($campaignId: String!) {
  deleteCampaign(campaignId: $campaignId)
}

# Anonymize a campaign for export
mutation AnonymizeCampaign($campaignId: String!, $options: AnonymizationInput!) {
  anonymizeCampaign(campaignId: $campaignId, anonymizeOptions: $options)
}
```

### Annotation mutations

```graphql
# Add a new annotation to a group of commands
mutation AddAnnotation(
  $campaignId: String!
  $commandIds: [String!]!
  $text: String!
  $tags: [String!]!
  $user: String!
  $favorite: Boolean
) {
  addCommandGroupAnnotation(
    campaignId: $campaignId
    commandIds: $commandIds
    text: $text
    tags: $tags
    user: $user
    favorite: $favorite
  ) {
    id
    text
    date
    tags {
      text
    }
  }
}

# Update an existing annotation
mutation UpdateAnnotation(
  $campaignId: String!
  $annotationId: String!
  $text: String!
  $tags: [String!]!
  $user: String!
) {
  updateAnnotation(
    campaignId: $campaignId
    annotationId: $annotationId
    text: $text
    tags: $tags
    user: $user
  ) {
    id
    text
  }
}

# Delete an annotation
mutation DeleteAnnotation($campaignId: String!, $annotationId: String!) {
  deleteAnnotation(campaignId: $campaignId, annotationId: $annotationId) {
    id
  }
}
```

### Beacon and host mutations

```graphql
# Hide one or more beacons
mutation HideBeacons($campaignId: String!, $beaconIds: [String!]!) {

    hidden
```

## Key enums

| Enum | Values |
|---|---|
| `BeaconType` | `dns`, `http`, `https`, `smb` |
| `BeaconLineType` | `CHECKIN`, `ERROR`, `INDICATOR`, `INPUT`, `METADATA`, `MODE`, `OUTPUT`, `TASK` |
| `MitreTechniques` | `Collection`, `CommandAndControl`, `CredentialAccess`, `DefenseEvasion`, `Discovery`, `Execution`, `Exfiltration`, `Impact`, `InitialAccess`, `LateralMovement`, `Persistence`, `PrivilegeEscalation`, `Reconnaissance`, `ResourceDevelopment`, `GoldenTicket` |
| `NodeColors` | `default`, `red`, `orange`, `gold`, `green`, `lime`, `forest`, `turquoise`, `indigo`, `violet`, `rose`, `vermilion` |
| `Shapes` | `circle`, `square`, `diamond`, `triangleUp`, `triangleDown`, `hexagonUp`, `hexagonDown`, `pentagonUp`, `pentagonDown` |
| `ParsingStatus` | `PARSING_NOT_STARTED`, `PARSING_QUEUED`, `PARSING_IN_PROGRESS`, `PARSING_COMPLETED`, `PARSING_FAILURE`, `NOT_READY_TO_PARSE`, `LIVE_PARSING_CS` |

## Scalar types

`DateTime` is a custom scalar representing an ISO 8601 date-time string (e.g. `"2020-08-17T19:37:41.000Z"`).
